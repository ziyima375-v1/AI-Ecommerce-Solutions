### 阶段 1：筑基战役 —— SAA-C03
**全称：** AWS Certified Solutions Architect – Associate
**目的：** 搞懂“云”是怎么回事。没有它，你的 AI 模型只能在本地跑，上不了天。
**周期：** 2025.12.16 - 2026.03.01（约 2.5 个月）

#### 📅 详细周计划

**第一板块：核心服务扫盲（12.16 - 01.15）**
*   **Week 1: IAM & EC2 (身份与计算)**
    *   *学什么：* 怎么配用户权限（IAM），怎么开虚拟机（EC2），怎么选机型（CPU vs GPU）。
    *   *你的结合点：* 想象你要把你的 Python 脚本放到云上跑，得选什么样的 EC2？
*   **Week 2: VPC (网络 - 最大的拦路虎)**
    *   *学什么：* 子网、路由表、网关、安全组。
    *   *难点：* 这是非科班最头疼的地方。
    *   *死命令：* **必须亲手在 AWS 控制台搭建一个 VPC**。如果不理解网络，以后你的模型连不上数据库。
*   **Week 3: S3 & Storage (存储)**
    *   *学什么：* S3（存图片/模型权重）、EBS（硬盘）、EFS。
    *   *你的结合点：* 你的 SDXL 模型权重文件、TRO 的截图数据，全部要存 S3。
*   **Week 4: Database (数据库)**
    *   *学什么：* RDS (SQL), DynamoDB (NoSQL)。
    *   *你的结合点：* TRO 系统的风控规则和历史记录存在哪里？

**第二板块：架构模式与高可用（01.16 - 02.05）**
*   **Week 5: HA & Scaling (高可用与扩容)**
    *   *学什么：* ELB (负载均衡), ASG (自动扩缩容)。
    *   *场景：* 假如你的 TRO 系统突然有 1 万个请求涌入，ASG 怎么自动加机器？
*   **Week 6: Serverless (无服务器)**
    *   *学什么：* **Lambda** (必考), API Gateway。
    *   *重要性：* 很多轻量级 AI 推理（如简单的 OCR）直接用 Lambda 最省钱。
*   **Week 7: Decoupling & Security (解耦与安全)**
    *   *学什么：* SQS/SNS (消息队列), KMS (加密)。

**第三板块：刷题冲刺（02.06 - 02.28）**
*   **Week 8-10: 题海战术**
    *   **工具：** Tutorials Dojo (TD)。
    *   **方法：**
        1.  先做 "Review Mode"，做一题看一题解析。
        2.  **错题本：** 极其重要！搞懂为什么选 A 不选 B。
        3.  做 "Timed Mode" 模拟考。
    *   **出关标准：** 连续 3 次模考分数 **> 85%**。

#### 🎯 SAA 考试日（建议：2026.03.01 周日）
*   **地点：** 推荐去线下的 **Pearson VUE 考试中心**（比在家考省心，不用担心网速和环境）。

### 阶段 2：升维战役 —— MLA-C01
**全称：** AWS Certified Machine Learning Engineer - Associate
**目的：** 证明你是“懂 AI 落地”的工程师，而不仅仅是“调包侠”。
**周期：** 2026.03.05 - 2026.06.20（约 3.5 个月）

#### 📅 详细周计划

**第一板块：SageMaker 深度工程化（03.05 - 04.05）**
*   **Week 1-2: SageMaker Training & Tuning**
    *   *学什么：* 怎么用 Docker 打包你的代码？怎么用 Spot Instances 省钱训练？怎么自动调参？
    *   *结合点：* 把你的 `tro_risk_analyzer.py` 想象成要在大规模集群上跑。
*   **Week 3-4: SageMaker Deployment (部署)**
    *   *学什么：* Real-time Endpoint (实时), Serverless Inference, Async Inference。
    *   *核心：* 面试必问——“模型太大了怎么部署？流量波动大怎么部署？”

**第二板块：GenAI 与 Bedrock（04.06 - 05.05）—— 重中之重**
*   **Week 5-6: Amazon Bedrock**
    *   *学什么：* 怎么调 Claude 3 / Llama 3？怎么做微调 (Fine-tuning)？
    *   *新考点：* **Guardrails** (如何防止模型说脏话/泄密) —— 这对你的 TRO 风控极其重要。
*   **Week 7-8: RAG & Agents**
    *   *学什么：* Knowledge Bases (知识库), Vector Database (向量数据库), Agents (智能体)。
    *   *实战：* 你的 GitHub 项目里应该加一个“基于 Bedrock 的跨境电商客服机器人”Demo。

**第三板块：MLOps 与数据流（05.06 - 05.31）**
*   **Week 9: Data Engineering**
    *   *学什么：* Kinesis (流数据), Glue (ETL), Athena。
    *   *场景：* 实时抓取电商数据并清洗。
*   **Week 10-11: MLOps**
    *   *学什么：* SageMaker Pipelines, Model Registry, Model Monitor (监控模型漂移)。
    *   *意义：* 这就是“小作坊”和“大厂”的区别。

**第四板块：终极冲刺（06.01 - 06.19）**
*   **Week 12-14: 针对性刷题**
    *   由于 MLA 是新考试，重点看 AWS 官方给的 **Sample Questions** 和 Udemy 上的最新模拟题。
    *   重点攻克：**GenAI 场景题** 和 **SageMaker 部署策略题**。

*   **时长：** 130 + 30 = 160 分钟。

### ⚠️ 给你的特别锦囊

1.  **关于英语：**
    *   AWS 考试的英语其实是“技术英语”。你不需要懂复杂的语法，你只需要看懂关键词：**"Most cost-effective" (最省钱)**, **"Highly available" (高可用)**, **"Lowest latency" (最低延迟)**。
    *   看到这些词，你就知道考点在哪里了。

2.  **关于实战：**
    *   不要只看书！你的优势是动手能力强。
    *   学到 S3 时，真的去写个 Python 脚本上传文件。
    *   学到 Bedrock 时，真的去调一下 API。

    *   SAA 的 VPC 阶段（第2-3周）你会很痛苦，感觉听天书。**这是正常的**，撑过去就豁然开朗。
